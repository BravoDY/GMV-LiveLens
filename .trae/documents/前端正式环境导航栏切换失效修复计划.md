# 前端正式环境导航栏无法切换 — 排查与修复计划

## 一、现象

- **位置**: 正式环境主页面 `/`（`frontend/index.html` 加载 `app.js`）
- **症状**: 顶部导航栏「实时看板」「采集配置」「任务管理」三个按钮点击无反应，视图不切换
- **复现条件**: 100% 必现

## 二、影响范围

| 页面 | 路由 | 是否影响 |
|------|------|----------|
| 正式环境主页 | `/` | ✅ 导航按钮可见但点击无效果 |
| 公开看板 | `/dashboard` | ⚠️ 导航栏被 `applyPublicDashboardMode()` 隐藏 (设计如此) |
| 测试看板 | `/dashboard-test` | ❌ 不影响 (独立的 `test-dashboard/app.js`) |

## 三、根因分析

### 证据链

1. **`switchView()` 存在但从未被导航栏按钮触发**

   [core.js:L291-L294](file:///c:/Users/yjd22/Desktop/python项目/GMV-LiveLens/frontend/core.js#L291-L294):
   ```javascript
   function switchView(view) {
     document.querySelectorAll(".view").forEach((el) => el.classList.toggle("active", el.id === view));
     document.querySelectorAll(".tab, .nav-item").forEach((el) => el.classList.toggle("active", el.dataset.view === view));
   }
   ```

2. **全项目搜索 `switchView` 调用点**：只在 [config.js:L1079](file:///c:/Users/yjd22/Desktop/python项目/GMV-LiveLens/frontend/config.js#L1079) 被调用一次（选择任务进入编辑时），**不存在任何点击事件绑定**。

3. **导航栏 HTML 无 onclick 属性**：
   ```html
   <button class="tab nav-item active" data-view="dashboard">实时看板</button>
   <button class="tab nav-item" data-view="config">采集配置</button>
   <button class="tab nav-item" data-view="manager">任务管理</button>
   ```

4. **`core.js` 中无任何 tab/nav-item 相关的事件监听器**。

5. **`app.js` 启动逻辑**：
   - `/`路径 → `shouldStartInternalDashboard()` → `startInternalDashboard()` — 只初始化数据，不绑定导航事件
   - `/dashboard` 路径 → `startPublicDashboardPolling()` → `applyPublicDashboardMode()` — 直接隐藏 `.header-nav`

### 根因结论

**导航栏按钮从未被绑定点击事件**。`switchView()` 函数存在于 `core.js` 中，但没有任何代码将 tab 按钮的 `click` 事件连接到 `switchView()`。这是一个**执行路径缺失** bug——`switchView()` 只在 `config.js` 中被程序化调用（进入任务编辑时切换到 config 视图），导航栏按钮本身从未触发它。

### 为什么会发生

`config.js` 能调用 `switchView("config")` 是因为它有明确的业务触发点（选择任务→编辑），而导航栏是"全局 UI 导航"，开发者可能遗漏了事件绑定，或者事件绑定代码在某次重构中被意外移除。

## 四、修复方案

**在 `frontend/app.js` 的 `startInternalDashboard()` 函数中添加导航栏委托点击事件绑定**。

### 修改文件

| 文件 | 修改内容 |
|------|----------|
| `frontend/app.js` | `startInternalDashboard()` 中新增 `.header-nav` 的 click 委托事件，使 `data-view` 按钮触发 `switchView()` |

### 实现代码

```javascript
// 在 startInternalDashboard() 函数末尾添加：
document.querySelector(".header-nav")?.addEventListener("click", (e) => {
  const btn = e.target.closest("[data-view]");
  if (!btn) return;
  const view = btn.dataset.view;
  if (view) switchView(view);
});
```

### 为什么放在 `app.js` 中

1. `startInternalDashboard()` 是主页（`/`）的启动入口，在这里绑定时机正确
2. `app.js` 已经负责页面初始化逻辑，新增导航事件绑定语义一致
3. 不影响 `/dashboard`（公开看板）和 `/dashboard-test`（测试看板），它们各有独立启动逻辑

### 为什么用事件委托

1. 不需要为每个按钮单独绑定
2. 之后新增 `data-view` 按钮自动生效
3. 代码量最小

## 五、风险评估

| 风险 | 等级 | 说明 |
|------|------|------|
| 影响公开看板 | 无 | `/dashboard` 路由不会调用 `startInternalDashboard()`，且 `.header-nav` 已被隐藏 |
| 影响测试看板 | 无 | `/dashboard-test` 使用独立的 `test-dashboard/app.js` |
| 影响已有功能 | 极低 | 仅新增事件绑定，不删除任何代码 |
| 性能影响 | 无 | 委托事件，只有一个监听器 |

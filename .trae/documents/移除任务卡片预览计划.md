# 移除任务管理卡片预览功能计划

## 1. 目标与范围
根据您的要求，将“任务管理 - 卡片管理”视图中每个任务的“实时预览”部分完全移除，以使卡片更加紧凑，同时去除不必要的视觉干扰。

**范围**：
- `frontend/app.js`（移除 `renderManager` 中卡片视图下的预览 DOM 生成代码）
- `frontend/styles.css`（调整 `.manager-card-body` 的 Grid 布局以适应移除图片后的单列结构）

## 2. 当前状态分析
- 目前 `frontend/app.js` 的 `renderManager()` 函数中，在卡片的 `<div class="manager-card-body">` 内部，除了文本信息区外，还会调用 `${renderPagePreview(task, "card")}` 来渲染右侧的实时缩略图。
- 对应的样式 `.manager-card-body` 使用了双列 Grid 布局：`grid-template-columns: minmax(0, 1fr) minmax(220px, 42%);`，左边是文字信息，右边是预览图。
- 您的截图中也清晰地圈出了这块不需要的区域。

## 3. 拟议的具体修改

### 3.1 frontend/app.js
- 找到 `renderManager()` 函数中的 HTML 模板。
- 将 `<div class="manager-card-body">` 内部的 `${renderPagePreview(task, "card")}` 完全删除。
- 保持“页面监控”视图（`renderManagerMonitor`）不变，因为您明确指定的是在“卡片管理”中移除。

### 3.2 frontend/styles.css
- 找到 `.manager-card-body` 类。
- 将 `grid-template-columns: minmax(0, 1fr) minmax(220px, 42%);` 改为 `display: block;` 或 `grid-template-columns: 1fr;`，让剩下的信息区域占满整个卡片宽度。

## 4. 验证步骤
1. 打开浏览器进入“任务管理”视图，确认卡片模式下每个任务卡片右侧的截图预览框已经消失。
2. 确认卡片内部的其他文字信息（如金额、巡检状态、Edge 状态等）排版正常且不会错位。

# 清理重复任务计划

## 1. 目标与范围
解决看板上出现两个相同名称（或幽灵任务）的“迪桑特京东官方旗舰店”的问题，同时找出没有在 `shops.csv` 中配置却默认出现的京东店铺。

**范围**：
- `data/gmv_livelens.sqlite3` 中的 `capture_tasks` 表数据清理。

## 2. 当前状态分析
根据数据库查询结果：
```python
[{'id': 39, 'platform': '京东', 'shop_name': '迪桑特京东官方旗舰店'}, 
 {'id': 54, 'platform': '京东', 'shop_name': '迪桑特京东官方旗舰店'}, 
 {'id': 55, 'platform': '京东', 'shop_name': 'DESCENTE京东运动旗舰店'}, 
 {'id': 56, 'platform': '京东', 'shop_name': '迪桑特京东自营店'}]
```
- **重复店铺**：确实存在两个 `迪桑特京东官方旗舰店`，它们的 ID 分别为 39 和 54。这是因为您或之前的自动化测试可能在前端手动“新建任务”（或旧的调试数据残留）创建了一个 ID 为 39 的同名店铺。而 ID 为 54, 55, 56 的店铺是后来通过 `shops.csv` 自动同步进去的。
- **为什么有一个默认京东店铺？** 您在截图左下角看到的“未连接”状态的“迪桑特京东官方旗舰店”，很可能就是那个旧的、没有跑起来的手动创建的任务 (ID: 39)。由于它的名字和 CSV 同步进来的完全一样，所以在看板上会同时显示两个名字一模一样的卡片。

## 3. 拟议的具体修改
我们不需要修改代码。只需要将数据库中多余的（或通过前端误添加的）旧任务数据清理掉即可。
为了安全起见，我们将删除 `id < 54` 且平台为京东的旧残留数据。具体操作：
- 通过 Python 脚本执行 SQL 语句 `DELETE FROM capture_tasks WHERE id = 39`，直接删除那条幽灵任务。

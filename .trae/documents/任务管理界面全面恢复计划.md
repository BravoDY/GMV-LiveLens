## 二、修复方案

### 2.1 文件变更清单

| 文件 | 修改内容 |
|------|----------|
| `backend/routers/common.py` | `build_snapshot()` 按 `sort_order` 排序 tasks |
| `frontend/dashboard.js` | `renderManagerGrid()` 新增平台操作按钮 + 新增 `renderManagerViewToggle()` + 筛选逻辑 + 卡片样式微调 |
| `frontend/index.html` | JS 版本号更新 |

### 2.2 各模块详细设计

#### 2.2.1 后端排序修复 (`common.py`)

`build_snapshot()` 中 tasks 增加 `sort_order` 排序：

```python
tasks.sort(key=lambda t: (t.get("sort_order") or 0, str(t.get("shop_name") or "")))
```

**理由**: `sort_order` = shops.csv 行索引（`enumerate`），先按 CSV 行顺序排，平台自然聚合（同平台店相邻），同 sort_order 按 shop_name 兜底。

#### 2.2.2 平台操作按钮 (`dashboard.js`)

在每个 `manager-platform-section` 的 `manager-platform-head` 中添加 `.manager-platform-actions` 区域，渲染 4 个按钮：

- **启动 Edge** → `callPlatformEdgeAction(platform, "launch")`
- **显示** → `callPlatformEdgeAction(platform, "show")`
- **隐藏** → `callPlatformEdgeAction(platform, "hide")`
- **关闭** → `callPlatformEdgeAction(platform, "close")`

按钮样式复用已有的 `.manager-btn` / `.manager-platform-actions` CSS。

点击后显示 loading 状态，完成后刷新卡片数据。

#### 2.2.3 视图切换 (`renderManagerViewToggle`)

在 `#managerViewToggle` 中渲染视图切换 UI：
- 当前已在 `renderDashboard()` 中渲染 managerGrid 的卡片视图

#### 2.2.4 状态筛选

为 `#statusFilters` 中的芯片绑定 click 事件，通过 `data-status` 筛选：
- `all` → 显示全部
- `ok` → 仅显示 `status === "ok"` 的卡片
- `alert` → 显示 status 非 ok 非 paused 的卡片
- `paused` → 仅显示 `status === "paused"` 的卡片

#### 2.2.5 前端排序保留

不修改 `sortByConfiguredOrder()`（该函数用于实时看板的平台级排序，逻辑不同）。任务管理视图直接使用后端 `build_snapshot()` 返回的 tasks 顺序（已按 sort_order 排好）。

## 四、验证方案

1. 启动服务，切换到「任务管理」视图
2. 确认每个平台标题右侧有「启动 Edge」「显示」「隐藏」「关闭」按钮
3. 点击某个按钮，验证后端日志有对应请求
4. 确认卡片顺序与 shops.csv 中行顺序一致（天猫旗下店铺在第一组）
5. 点击筛选芯片「告警」，仅显示异常状态卡片
6. 点击卡片，验证可跳转到采集配置编辑

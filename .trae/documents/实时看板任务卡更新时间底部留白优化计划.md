# 实时看板任务卡更新时间底部留白优化计划

## Summary
- 目标：优化“实时看板”中店铺任务卡底部的视觉留白，让 `更新时间` 不再贴近下边框，整体观感更舒展。
- 成功标准：店铺任务卡底部留白略微增加，视觉更平衡；不改变卡片信息层级，不引入布局错位，不影响顶部闪光、数字高亮或告警态。
- 范围：仅处理实时看板店铺任务卡 `store-card` 内的 `更新时间` 与卡片底部间距关系。

## Current State Analysis
- `frontend/dashboard.js`
  - 店铺任务卡结构固定，`更新时间` 由 `<div class="updated-at">更新时间 ...</div>` 渲染，位于卡片最后一行。
- `frontend/styles.css`
  - 店铺卡样式定义在后段的 `.store-card` 规则中，当前 `padding` 为 `12px 16px 10px`，底部内边距仅 `10px`。
  - `.updated-at` 当前仅定义了 `font-size: 12px; line-height: 1.2; color: #7f8da3;`，没有额外的下方留白控制。
- 结论
  - 当前“贴近下边框”的主要原因是卡片底部内边距偏紧，而不是 DOM 结构问题。
  - 这是典型的 L1 轻量 UI 优化，宜做最小改动，不调整 JS，不改卡片高度与内容结构。

## Proposed Changes
- `frontend/styles.css`
  - 修改 `.store-card`
    - 将底部内边距从 `10px` 小幅增加到更舒适的值。
    - 推荐优先方案：仅微调 `padding` 的第三个值，例如从 `12px 16px 10px` 调整为 `12px 16px 13px` 或同级别的小幅提升。
    - 原因：直接增加底部呼吸感，影响面最小，不会单独改变文字排版节奏。
  - 视实际效果决定是否微调 `.updated-at`
    - 若仅调整卡片底部内边距仍显偏紧，可给 `.updated-at` 增加极小的 `padding-bottom` 或略调 `line-height`。
    - 原因：作为次选手段，只在需要时补细节，不优先使用，避免双重叠加导致底部过空。
- 不修改的文件
  - `frontend/dashboard.js` 不改：结构已正确，问题属于纯样式层。
  - 其它卡片样式不改：如 `total-card`、`platform-summary-card`、占位卡，避免无关联动。

## Assumptions & Decisions
- 决策：本次按“稍微放宽一点点，遵循美感”执行，采用保守微调，不做明显增高。
- 决策：优先通过 `.store-card` 底部内边距解决，因为它更符合“卡片内容与边框关系”的根因。
- 假设：店铺卡高度 `148px` 在当前信息密度下足够承载这次几像素级的留白增加，不会导致内容挤压。
- 假设：顶部边框闪光层、数字高亮、告警态边框都不会因底部留白的小幅变化受影响。

## Verification Steps
- 静态检查
  - 确认只改动 `frontend/styles.css` 中店铺卡相关规则。
  - 确认未影响 `.updated-at` 的颜色、字号与层级语义。
- 页面检查
  - 进入“实时看板”，观察多张店铺任务卡底部视觉留白是否更均衡。
  - 对比调整前后，确认 `更新时间` 不再显得贴底，但也不会显得悬空。
  - 检查普通态、悬浮态、`is-flashing` 状态、`is-alert` 状态下卡片底部观感是否一致。
- 回归关注点
  - 卡片高度不变时，确认目标、进度条、更新时间没有发生重叠或压缩。
  - 确认占位卡与平台汇总卡未被联动改动。

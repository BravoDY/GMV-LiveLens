# 采集配置和任务管理优化计划

## 1. 需求与目标
- **完全移除采集配置中的“步骤1 初始化任务”**：清理所有相关的 UI 元素、事件绑定和流程引导文案。
- **平台与店铺名严格对应 `shops.csv`**：在任务管理（卡片管理、页面监控）和实时看板中，杜绝手动预设平台和店铺。如果数据库中存在但 `shops.csv` 中未填报的店铺/平台，前端不予呈现；同时取消对平台名称的强制归一化（如“淘宝”强转“天猫”），确保完全按 CSV 的字面意义显示。

## 2. 当前状态分析
- **初始化任务 UI**：`frontend/index.html` 中存在 `#setupStepInit` 区块，`frontend/config.js` 和 `frontend/app.js` 中有对应的逻辑（如 `initShops` 函数）和思维导图引导。
- **列表呈现逻辑**：`managerFilteredTasks` 和 `liveTasks` 目前基于 `state.snapshot.tasks` 渲染，只要数据库里有就会显示，未严格校验其是否存在于 `state.shopConfigs`（由 `shops.csv` 解析而来）中。
- **平台硬编码**：`frontend/dashboard.js` 中硬编码了 `PLATFORM_ORDER`，`frontend/core.js` 中的 `normalizePlatform` 强制将未知平台归类为“其他平台”或进行别名映射。

## 3. 具体修改步骤

### 步骤 3.1 移除“初始化任务”相关 UI 和逻辑
- **文件 `frontend/index.html`**：
  - 删除 `<div class="setup-step-card" id="setupStepInit">...</div>` 整个 DOM 节点。
  - 将 `#setupCurrentMeta` 的默认文本修改为：“请从任务管理选择店铺进入编辑。”。
  - 将 `#setupNextStep` 的默认文本修改为：“下一步：请从任务管理选择要配置的店铺。”。
- **文件 `frontend/config.js`**：
  - 删除 `initShops` 异步函数。
  - 在 `renderSetupWorkbench` 函数中，删除对 `initCard` 的引用。
  - 在 `setupFlowState` 函数中，修改 `!task` 时的 `nextStep` 提示，移除关于“一键初始化”的说明，改为“请先在任务管理中选中一个任务，或进入当前待配置店铺。”。
- **文件 `frontend/app.js`**：
  - 删除底部 `$("initShops")?.addEventListener(...)` 的事件绑定。
  - 在 `renderConfigFlowPage` 函数中，从 `mainSteps` 数组中移除 `STEP 01 初始化任务` 对象，并确保后续步骤的序号与展示逻辑正常。

### 步骤 3.2 严格按照 `shops.csv` 过滤呈现数据
- **文件 `frontend/core.js`**：
  - 修改 `liveTasks()`，增加 `shopConfigForTask(task)` 校验，只返回存在于配置中的任务：
    ```javascript
    function liveTasks() {
      return (state.snapshot.tasks || []).filter((task) => task.enabled && shopConfigForTask(task));
    }
    ```
  - 修改 `normalizePlatform(platform)`，移除所有别名映射与“其他平台”兜底，直接返回原字符串（去除两端空格），确保完全意义对应：
    ```javascript
    function normalizePlatform(platform) {
      return String(platform || "未知平台").trim();
    }
    ```
- **文件 `frontend/app.js`**：
  - 修改 `managerFilteredTasks()`，在过滤条件的最前面增加拦截：
    ```javascript
    if (!shopConfigForTask(task)) return false;
    ```
    从而确保不在 `shops.csv` 中的幽灵任务或废弃任务绝对不会在任务管理中呈现。

### 步骤 3.3 移除预设平台顺序，支持动态平台
- **文件 `frontend/dashboard.js`**：
  - 删除顶部的 `const PLATFORM_ORDER = ["天猫", "京东", "得物", "抖音", "唯品会"];`。
  - 在 `buildDashboardViewModel` 中，当匹配不到 `platformMeta` 时，动态生成 fallback 的图标和样式：
    ```javascript
    const meta = platformMeta[platform] || { 
      label: platform, 
      icon: platform.charAt(0), 
      color: "var(--color-other)", 
      glow: "rgba(142, 160, 184, 0.22)" 
    };
    ```
  - 修改 `platforms.sort` 的排序依据，使用 `frontend/core.js` 中基于 `shops.csv` 动态生成的 `platformOrder()`：
    ```javascript
    const order = platformOrder();
    platforms.sort((a, b) => {
      const ai = order.indexOf(a.platform);
      const bi = order.indexOf(b.platform);
      return (ai === -1 ? 999 : ai) - (bi === -1 ? 999 : bi);
    });
    ```

## 4. 测试与验证标准
1. **界面检查**：进入“采集配置”页面，确认“步骤1. 初始化任务”已完全消失，导图说明正常，且无控制台报错。
2. **过滤检查**：在 SQLite 中手动插入一条 `shops.csv` 中不存在的测试任务，进入“任务管理”和“实时看板”，确认该任务**不会**被渲染。
3. **动态平台检查**：在 `shops.csv` 中新增一个未预设的平台（如“小红书”），刷新页面后，看板与任务管理应能正常将其分组渲染为“小红书”，并取第一个字“小”作为 Icon。

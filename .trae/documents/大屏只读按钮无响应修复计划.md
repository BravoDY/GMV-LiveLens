# 大屏只读按钮无响应修复计划

## 一、现象

采集配置页面底部「天猫专享大屏只读模式」区域的三个按钮：

* **开启只读** (`#startScreenReadonly`)

* **刷新只读结果** (`#refreshScreenReadonly`)

* **清空测试记录** (`#clearScreenReadonly`)

点击后完全无响应。

## 二、根因分析

与「启动采集」按钮 Bug 完全相同的模式：

1. **函数已存在** — [edge.js:L584-L614](file:///c:/Users/yjd22/Desktop/python项目/GMV-LiveLens/frontend/edge.js#L584-L614)：

   ```javascript
   async function startScreenReadonly() { ... }
   async function refreshScreenReadonly() { ... }
   async function clearScreenReadonly() { ... }
   ```

2. **HTML 按钮已存在** — [index.html:L187-L189](file:///c:/Users/yjd22/Desktop/python项目/GMV-LiveLens/frontend/index.html#L187-L189)：

   ```html
   <button id="startScreenReadonly" type="button">开启只读</button>
   <button id="refreshScreenReadonly" type="button">刷新只读结果</button>
   <button id="clearScreenReadonly" type="button">清空测试记录</button>
   ```

3. **但全局事件绑定区（core.js:L787-L794）缺失这三个按钮**。全局绑定区已注册了 `testOcr`、`saveTask`、`captureAllButton`、`schedulerToggle`、`previewRemotePage`、`resumeAfterLogin`，唯独没有 `startScreenReadonly` / `refreshScreenReadonly` / `clearScreenReadonly`。

### 根因结论

**三个屏幕只读按钮从未被绑定 click 事件**。这是和「启动采集」按钮完全相同的"函数存在但事件缺失" Bug。

## 三、修复方案

在 `frontend/core.js` 全局事件绑定区（L793 之后）新增 3 行事件绑定：

```javascript
$("startScreenReadonly")?.addEventListener("click", () => startScreenReadonly().catch((err) => showMessage(parseApiError(err), true)));
$("refreshScreenReadonly")?.addEventListener("click", () => refreshScreenReadonly().catch((err) => showMessage(parseApiError(err), true)));
$("clearScreenReadonly")?.addEventListener("click", () => clearScreenReadonly().catch((err) => showMessage(parseApiError(err), true)));
```

### 文件变更

| 文件                    | 变更                                 |
| --------------------- | ---------------------------------- |
| `frontend/core.js`    | 全局事件绑定区新增 3 行 screen readonly 按钮绑定 |
| `frontend/index.html` | core.js 版本号更新                      |

## 四、验证方案

1. 刷新浏览器 `/` 页面
2. 切换到「采集配置」
3. 确认已有 Edge 会话和页面绑定
4. 点击「开启只读」→ 按钮文字应变灰/loading → 调用 API → 下方显示 payAmt 结果
5. 点击「刷新只读结果」→ 再次请求 API 更新显示
6. 点击「清空测试记录」→ 记录区域清空

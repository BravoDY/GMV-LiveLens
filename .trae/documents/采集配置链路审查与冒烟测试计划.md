# 采集配置完整链路审查与冒烟测试计划

## 一、链路概览

```
任务管理卡片
  └─ click → loadTaskIntoConfig(task)
      └─ switchView("config")
        └─ renderSetupWorkbench()        ← 配置工作台UI
          ├─ [扫描页签] → scanBind() → GET /api/tasks/{id}/page-candidates
          ├─ [确认绑定] → confirmBind() → POST /api/tasks/{id}/rebind-page
          ├─ [生成预览] → previewRemotePage() → POST /api/edge-sessions/{sid}/pages/{pid}/preview
          ├─ [  OCR测试] → testOcr() → POST /api/test-ocr
          └─ [保存任务] → saveTask() → POST /api/tasks
                              ↓
实时看板 ← scheduler._capture_screen_readonly_once() ← scheduler._run_loop()
  └─ RemoteEdge.read_screen_pay_amount(page_id) ← CDP → Edge浏览器DOM
```

## 三、冒烟测试步骤

### 步骤 1：任务管理→采集配置
- 刷新 `/` 页面
- 切换到「任务管理」视图
- 点击任意一张任务卡片（如 天猫官旗店）
- **验证**：页面跳转到采集配置，`#setupCurrentName` 显示「天猫 / 天猫官旗店」

### 步骤 2：Edge 会话连接
- 在采集配置中检查 `bindSessionSelect` 下拉框
- **验证**：有可选的 Edge 会话（如 天猫_天猫官旗店）

### 步骤 3：页签扫描
- 点击「扫描页签」按钮
- **验证**：API 返回候选页签列表，UI 更新

### 步骤 4：OCR 预览
- 点击「生成预览」
- **验证**：右侧出现页面截图

### 步骤 5：保存任务
- 框选 GMV 区域后点击「保存任务」
- **验证**：显示成功消息，`state.snapshot` 更新

### 步骤 6：采集触发
- 回到实时看板，检查该店铺 GMV 值是否更新
- 点击「采集全部」或单次采集
- **验证**：后端日志出现 capture_once 调用

### 步骤 7：大屏只读
- `POST /api/tasks/{id}/capture-once`（value_source=screen_readonly）
- **验证**：返回 pay_amt 非空

### 步骤 8：页面无 JS 报错
- 打开浏览器 F12 Console
- 逐一执行上述操作
- **验证**：Console 无红色错误

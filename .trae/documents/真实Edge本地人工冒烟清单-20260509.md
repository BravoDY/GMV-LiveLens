# 真实 Edge 本地人工冒烟清单

> 适用阶段：稳定化验收、发版前回归、真实店铺环境变更后验证
>
> 适用前提：
> - Windows 本地环境可正常启动项目
> - 店铺登录态已保存在 `data/edge_profiles/`
> - 不删除、不迁移 `edge_profiles`、`sqlite3`、`screenshots`
> - 后端服务可通过 `python -m uvicorn backend.main:app --host 127.0.0.1 --port 8100` 启动

---

## 1. 执行前准备

### 1.1 启动前检查

| 检查项 | 操作 | 预期结果 |
|---|---|---|
| 后端可启动 | 启动 `uvicorn` | 控制台显示 `Application startup complete` |
| 前端主页可访问 | 打开 `http://127.0.0.1:8100/` | 页面正常加载，含 GMV 看板 UI |
| 店铺配置可读取 | 执行 `python tests/smoke_api.py` 或访问 `/api/shops` | 返回店铺配置列表 |
| Edge 会话列表可读取 | 访问 `/api/edge-sessions` | 返回会话列表，含默认会话或店铺会话 |

### 1.2 真实环境限制

- 执行前确认个人 Edge 中无未保存工作，因为某些会话启动/重绑场景可能影响受控 Edge 进程。
- 只在用户确认可操作的店铺环境中执行，不要用未知来源的登录态目录。
- 若出现平台风控、登录失效、验证码，不要继续自动重试，应立即记录并停止该链路验证。

## 2. Edge 四动作冒烟

### 2.1 启动 Edge

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 在工作台或对应 API 触发 `start-edge` / `launch-edge` | 返回成功状态，不是 accepted 占位结果 |
| 2 | 检查 Edge 窗口是否出现 | 出现受控业务窗口，不是空白残留窗口 |
| 3 | 访问 `/api/edge-sessions/{session_id}/health` | 返回 `debug_available=True` 或明确可解释状态 |

### 2.2 显示 / 隐藏 / 再显示

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 调用 `show-edge` | 目标窗口显示到前台或可见区域 |
| 2 | 调用 `hide-edge` | 目标窗口从桌面隐藏 |
| 3 | 再次调用 `show-edge` | 同一会话窗口可恢复显示，不出现“隐藏后无法恢复” |

### 2.3 关闭 Edge

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 调用 `close-edge` | 返回成功状态，并给出真实结果 |
| 2 | 检查调试端口 | 对应 debug port 不再占用 |
| 3 | 检查残留进程 | 没有残留受控 profile 的孤儿进程 |

### 2.4 异常记录

- 若 `show-edge` 成功返回但窗口不可见：记录会话 ID、端口、是否存在窗口句柄。
- 若 `close-edge` 后端口仍在：记录 PID、会话 ID、profile 路径。
- 若出现 accepted/处理中无后验收：视为失败。

## 3. 登录态与业务页恢复

### 3.1 登录态保留

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 启动指定店铺会话 | 进入已登录状态，不跳平台登录页 |
| 2 | 关闭会话 | 正常关闭受控 Edge |
| 3 | 再次启动同店铺会话 | 仍保留登录态 |

### 3.2 目标页恢复

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 让任务绑定到目标业务页 | 记录目标页 URL |
| 2 | 重启受控 Edge | 会话重建 |
| 3 | 检查自动恢复 | 可根据 `target_page_url` 自动重绑，必要时再人工重绑 |

### 3.3 失败判定

- 启动后直接落到平台登录页，且原本应保留登录态：失败。
- 会话重启后 `page_id` 失效但无法按目标 URL 恢复绑定：失败。

## 5. OCR 与采集链路

### 5.1 单次采集

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 对已绑定任务执行 `capture-once` | 返回采集结果 |
| 2 | 检查识别值 | 金额格式正常，来源说明正确 |

### 5.2 连续确认逻辑

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 在真实业务页连续触发采集 | 任务状态会经历 `pending_confirm -> ok` 或合理异常态 |
| 2 | 检查异常跳变 | 大幅异常值不应直接作为可信值写入 |

### 5.3 大屏只读

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 打开平台大屏只读页 | 页签定位到正确大屏 |
| 2 | 调用只读读取 | 能返回平台对应金额字段 |
| 3 | 与页面展示比对 | 数值口径一致或可解释 |

## 8. 失败时的最小排查顺序

1. 先看 `uvicorn` 控制台是否报错。
2. 再看 `/api/edge-sessions/{session_id}/health` 的返回结构。
3. 再确认调试端口是否占用、窗口是否存在、页签是否存在。
4. 再确认是否是登录态失效、平台风控、业务页改版，而不是代码回归。
5. 若失败与真实环境强相关，不要立即改业务代码，先记录现场。

## 10. 当前建议

- 发版前至少执行一次完整人工冒烟。
- 若只改动前端展示或文档，可缩小为 `WebSocket + 看板 + 任务状态一致性` 子集。
- 若改动 `remote_edge`、`scheduler`、`OCR`、`页签绑定`、`大屏只读` 任一链路，必须重跑完整人工清单。

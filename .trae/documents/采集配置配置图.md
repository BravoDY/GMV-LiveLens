# 采集配置配置图

## 1. 使用目标
- 这份文档用于说明`采集配置`页面的完整操作链路。
- 阅读目标：
  - 快速看懂当前应该怎么配置一个店铺任务。
  - 快速判断某一步失败后该去哪里处理。
  - 快速理解页面每个区域分别负责什么。

## 2. 全链路图
```text
进入采集配置
└─ 系统自动聚焦当前待配置店铺
   ├─ 没有待配置店铺
   │  └─ 查看“当前处理店铺”卡片提示
   └─ 有当前任务
      ├─ 步骤 1：初始化任务
      │  ├─ 首次进入项目
      │  │  └─ 点击「一键初始化默认店铺」
      │  └─ 初始化完成
      │     └─ 自动进入第一家待配置店铺
      │
      ├─ 步骤 2：选择当前任务页面
      │  ├─ 去任务管理
      │  │  └─ 启动 / 显示当前店铺对应 Edge
      │  ├─ 在后台中打开正确业务页面
      │  ├─ 回到采集配置
      │  ├─ 点击「扫描当前会话页签」
      │  ├─ 系统列出当前会话全部页签
      │  ├─ 手动点击「选择此页」
      │  └─ 点击「使用此页面」
      │
      ├─ 步骤 3：生成预览
      │  ├─ 点击「生成预览」
      │  ├─ 右侧出现页面截图
      │  └─ 若失败
      │     └─ 按提示回任务管理确认页签，再回来重新扫描
      │
      ├─ 步骤 4：框选 GMV
      │  ├─ 在右侧截图上拖拽框选 GMV 数字区域
      │  └─ 框选完成后进入测试/保存阶段
      │
      ├─ 步骤 5：测试识别
      │  ├─ 点击「测试识别」
      │  ├─ 查看 OCR 结果
      │  └─ 若不准
      │     └─ 重新框选，继续测试
      │
      └─ 步骤 6：保存并进入下一家
         ├─ 点击「保存并进入下一家」
         ├─ 系统保存当前任务页签绑定与框选区域
         └─ 自动切到下一家待配置店铺
```

## 3. 失败分支图
```text
A. 扫描不到页签
采集配置 -> 扫描当前会话页签
└─ 当前会话没有可绑定页签
   ├─ 去任务管理显示当前店铺 Edge
   ├─ 确认后台正确页面已打开
   └─ 回采集配置重新扫描

B. Edge 会话未就绪
采集配置 -> 扫描当前会话页签
└─ Edge 会话未接通
   ├─ 去任务管理点击「启动Edge」或「显示Edge」
   ├─ 确认窗口和页面都已打开
   └─ 回采集配置重新扫描

C. 已绑定页签失效
采集配置 -> 生成预览 / 正式采集
└─ remote_page_not_found
   ├─ 系统不会自动恢复
   ├─ 系统不会自动新开页
   ├─ 去任务管理确认正确页签仍打开
   └─ 回采集配置重新扫描并重新绑定

D. OCR 不准确
采集配置 -> 测试识别
└─ 结果不合理
   ├─ 保留当前绑定页签
   ├─ 重新拖拽框选 GMV 区域
   └─ 再次测试后保存
```

## 4. 产品结构图
```text
采集配置页面
├─ 左侧：配置工作台
│  ├─ 当前处理店铺卡片
│  │  ├─ 当前店铺是谁
│  │  ├─ 当前会话是谁
│  │  ├─ 当前状态是什么
│  │  └─ 下一步该做什么
│  ├─ 步骤 1：初始化任务
│  │  └─ 首次导入默认店铺任务
│  ├─ 步骤 2：选择当前任务页面
│  │  ├─ 选择 Edge 会话
│  │  ├─ 扫描当前会话页签
│  │  ├─ 查看页签列表
│  │  ├─ 手动选择页签
│  │  └─ 绑定到当前任务
│  └─ 步骤 3：保存并进入下一家
│     ├─ 清空当前标定重来
│     └─ 显示成功/失败提示
│
└─ 右侧：截图标定区
   ├─ 生成预览
   ├─ 拖拽框选 GMV
   ├─ 查看坐标
   ├─ 测试识别
   └─ 保存并进入下一家
```

## 5. 页面区域职责
### A. 当前处理店铺卡片
- 告诉你当前正在配置哪家店、对应哪个 Edge 会话、当前链路到哪一步。
- 如果你只看一个区域，优先看这里。

### B. 选择当前任务页面
- 这是“页签绑定区”。
- 只负责当前任务，不负责批量绑定其它任务。
- 这里只做三件事：
  - 扫描当前会话页签
  - 手动选一个页签
  - 绑定到当前任务

### C. 截图标定区
- 这是“预览 + 框选 + OCR 测试”区域。
- 前提是步骤 2 已经完成。
- 如果没有绑定页签，这里不应该成为你的第一操作区。

### D. 保存并进入下一家
- 当前任务完成后，统一从这里收尾。
- 目标不是停留在当前任务，而是快速推进到下一家待配置任务。

## 6. 推荐操作手册
### 第一次配置某家店
1. 去任务管理启动或显示当前店铺 Edge。
2. 在后台里打开你真正要 OCR 的页面。
3. 回采集配置，扫描当前会话页签。
4. 从列表中手动选择正确页签并绑定。
5. 生成预览，在右侧框选 GMV。
6. 测试识别。
7. 保存并进入下一家。

### 后续重新调整某家店
1. 进入采集配置并聚焦该店。
2. 如果只是框选不准，直接重新生成预览并重新框选。
3. 如果页签失效，先去任务管理确认正确页签，再回来重新扫描和重新绑定。

## 7. 当前设计原则
- 当前任务优先，不做批量复杂操作。
- 手工选择页签优先，不做偷偷自动恢复。
- 失败必须有提示，而且提示里必须包含下一步操作。
- 页面文案要像导航，不让用户自己猜流程。

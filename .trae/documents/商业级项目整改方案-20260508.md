### P1-1：添加 Lint/Formatter 配置

| 项目 | 详情 |
|---|---|
| **风险等级** | P1 |
| **涉及文件** | 新建 `pyproject.toml`、`.ruff.toml`（零业务代码修改） |
| **商业影响** | 无 lint 时多人协作代码风格必然分化，Code Review 成本指数增长 |
| **风险** | 首次运行 ruff 可能产生大量 warning，但不影响功能 |
| **收益** | 零成本统一代码风格、自动发现潜在 Bug、CI 可集成 |

**修改方式**（不改任何 .py 代码）

```toml
# pyproject.toml (新建)
[project]
name = "gmv-livelens"
version = "0.3.0"
requires-python = ">=3.11"

[tool.ruff]
line-length = 120
target-version = "py311"

[tool.ruff.lint]
select = ["E", "F", "W", "I", "N", "UP", "B", "C4"]
ignore = ["E501"]  # 行长度由 formatter 处理

[tool.ruff.format]
quote-style = "double"
```

```toml
# .ruff.toml (新建，覆盖默认行为)
exclude = [".venv", "__pycache__", ".idea", "data"]
```

### P1-2：拆分 main.py 路由

| 项目 | 详情 |
|---|---|
| **风险等级** | P1 |
| **涉及文件** | 新建 `backend/routers/` 目录，从 `backend/main.py` **剪切**函数到子模块 |
| **改动范围** | main.py 2347行 → ~200行（仅保留 app 初始化 + 工具函数） |
| **商业影响** | 所有路由放一个文件，新增 API 必然冲突，Code Review 无法并行 |
| **风险** | 拆分时 import 路径变更可能导致启动失败，需逐个验证 |
| **收益** | API 开发冲突率降至零，独立路由可单独测试 |

**修改方式**

```
backend/routers/              # 新建目录
├── __init__.py               # 空
├── tasks.py                  # ← 剪切 /api/tasks/* 路由 (约 500 行)
├── edge_sessions.py          # ← 剪切 /api/edge-sessions/* 路由 (约 850 行)
├── platforms.py              # ← 剪切 /api/platforms/* 路由 (约 120 行)
├── shops.py                  # ← 剪切 /api/shops/* 路由 (约 150 行)
├── ocr.py                    # ← 剪切 /api/test-ocr, /api/ocr/* (约 200 行)
└── system.py                 # ← 剪切 /api/settings, /api/scheduler, /api/windows (约 80 行)
```

**具体步骤**：

1. 创建 `backend/routers/__init__.py`
2. 将 main.py 中的路由函数剪切到对应子模块
3. main.py 中 `from backend.routers.tasks import router as tasks_router` + `app.include_router(tasks_router)`
4. 删除 main.py 中已空的路由函数（core.js 调用的逻辑不变）
5. 运行 `full_test.py --skip-api` 确认不报错
6. 启动服务跑 `smoke_api.py` 确认所有 API 正常

```python
from logging.handlers import RotatingFileHandler

LOG_DIR = ROOT_DIR / "logs"
LOG_DIR.mkdir(exist_ok=True)

logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s [%(levelname)s] %(name)s: %(message)s",
    handlers=[
        logging.StreamHandler(),
        RotatingFileHandler(
            LOG_DIR / "gmv_livelens.log",
            maxBytes=10 * 1024 * 1024,
            backupCount=5,
            encoding="utf-8",
        ),
    ],
)
```

### P2-2：添加 Docker 支持

| 项目 | 详情 |
|---|---|
| **风险等级** | P1（部署） |
| **涉及文件** | 新建 `Dockerfile`、`docker-compose.yml`、`.dockerignore` |
| **商业影响** | 无容器化意味着环境不可控，无法在生产服务器上可靠启动 |
| **风险** | Windows 容器与 Linux 容器差异（本项目 Windows ONLY） |
| **收益** | 一键部署，环境完全可复现 |

```dockerfile
# Dockerfile (Windows Container)
FROM python:3.11-windowsservercore
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
RUN python -m playwright install chromium
COPY backend/ ./backend/
COPY frontend/ ./frontend/
COPY data/shops.csv ./data/
EXPOSE 8100
CMD ["python", "-m", "uvicorn", "backend.main:app", "--host", "0.0.0.0", "--port", "8100"]
```

```yaml
      - uses: actions/setup-python@v5
```

# 去除采集暂停功能计划

## 1. 目标与范围
确认“采集全部”和“采集暂停”功能是否添加成功且链路跑通，并根据您的最新需求，移除界面上不需要的“采集暂停”按钮及其相关逻辑。

**范围**：
- `frontend/index.html`（移除按钮元素）
- `frontend/core.js`（移除绑定的点击事件和执行逻辑函数）

## 2. 当前状态分析
- 经过检查，`captureAllTasks` 和 `pauseAllTasks` 函数都已成功在 `frontend/core.js` 中实现，它们会并行向所有活动的任务分别发送 `/api/tasks/{id}/capture-once` 和 `/api/tasks/{id}/enabled`（设为 false）的 API 请求，逻辑跑通且事件已正确绑定。
- 在 `frontend/index.html` 的 header-actions 区域，这两个按钮已经存在。
- 现在您希望只保留“采集全部”，去掉“采集暂停”。

## 3. 拟议的具体修改

### 3.1 frontend/index.html
- **移除按钮**：找到 `id="pauseAllButton"` 的 `<button>` 元素并将其从 DOM 树中删除。保留 `id="captureAllButton"`。

### 3.2 frontend/core.js
- **移除事件绑定**：在文件末尾移除 `$("pauseAllButton")?.addEventListener("click", () => pauseAllTasks());` 这一行。
- **移除执行逻辑**：删除 `async function pauseAllTasks() { ... }` 的完整函数定义，以保持代码整洁。

## 4. 验证步骤
1. 打开浏览器检查看板顶部导航栏，“采集暂停”按钮应已消失，且“采集全部”按钮依然存在。
2. 确保页面的其他全局事件（如点击采集全部等）依然正常工作，没有因为移除 JS 代码而产生报错。

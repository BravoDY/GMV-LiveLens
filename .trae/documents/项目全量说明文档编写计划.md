## 2026-05-03 最新项目状态
- Edge 四按钮双窗口和随机多标签问题已经完成代码收口，并通过真实 Edge 冒烟测试。
- 两个原始问题的最终结论：
  - “只启动第一个任务的 Edge 却出现两个浏览器窗口”的历史原因是店铺级前端曾执行 `start -> show` 双调用，后端 `show` 找不到窗口时又可能补开 native window。当前已改为店铺级和平台级“启动Edge”统一走后端 `show_edge` 单入口，未运行时由后端可见启动，已运行时只恢复窗口。
  - “启动 N 个 Edge 后每个 Edge 随机出现多个网页标签”的历史原因是后端启动时固定先开 `edge://newtab/`，随后 `_ensure_launch_page()` 再打开目标页，Edge 历史恢复页/新标签页也可能一起留下。当前已改为优先直接打开目标 URL；没有目标 URL 时只保留一个空白页或 Edge 自带新标签页，并在每次 `start/show/launch` 后执行严格单页清理。
- 当前 Edge 四按钮语义：
  - 店铺级“启动Edge”= 启动并显示，前端直接调用后端 `show` 单入口，不再 `start` 后再 `show`。
  - 店铺级“显示Edge”= 恢复/显示当前会话；若未运行，可由后端启动并显示。
  - 平台级“启动Edge”/`launch-edge`/`start-edge` = 对平台内店铺按顺序启动并显示。
  - 平台级“显示Edge”= 只恢复已运行窗口，不自动启动未运行会话。
  - “隐藏Edge”= 优先移到屏幕外，其次最小化/隐藏，尽量保持 CDP 截图能力。
  - “关闭Edge”= 优雅关闭后强杀兜底，确保对应 profile/端口进程消失。
- 当前四按钮返回结构已统一补齐：
  - `action`
  - `stage`
  - `reason_code`
  - `window_action`
  - `window_diagnostics`
  - `recovery_attempted`
  - `debug_available`
  - `window_found`
  - `closed`
  - `page_count`
  - `primary_page_url`
  - `closed_extra_pages_count`
- 已新增真实冒烟测试脚本：
  - `backend/tools/smoke_edge_buttons.py`
  - 该脚本会真实执行启动、显示、隐藏、关闭动作；运行前必须提醒会真实关闭受控 Edge。
- 已执行的真实烟测结论：
  - 命令：`.venv\Scripts\python.exe backend\tools\smoke_edge_buttons.py --base-url http://127.0.0.1:8100 --live --loops 3 --include-platform`
  - 单店铺冷启动、连续启动/显示 3 次、隐藏->显示 3 轮、关闭、关闭后重启均 PASS。
  - 京东平台 3 个店铺的启动、显示、隐藏、隐藏后显示、关闭均 PASS。
  - 最终 `failures=0`。

# 全渠道实时GMV 卡片增加同比数据 — 设计审查与优化方案

## 设计审查：当前卡片结构

```
┌─ total-card-shell ─────────────────────────────────────┐
│ 北京时间：18:15:21                                      │
│ ┌─ total-card (青色边框) ──────────────────────────────┐│
│ │ 全渠道实时GMV                          · 实时模式    ││
│ │ ¥280,530,340   ← font-size: 52px / 900 weight        ││
│ │                                                      ││
│ │  总目标                 总达成进度                   ││
│ │  ¥230,910,000  (18px)   12.1%  (36px)                ││
│ │                                                      ││
│ │ ██████░░░░░░░░░░░░░░░░░░  (progress bar)             ││
│ └──────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────┘
```

## 设计问题分析

### 问题 1：直接 3 列 grid 会破坏视觉节奏

当前 `grid-template-columns: 1fr 1fr`，2 列布局：
- 左列：总目标（绝对值 ¥230,910,000，18px 字号）
- 右列：总达成进度（百分比 12.1%，**36px** 大字重）

如果改为 `1fr 1fr 1fr` 3 列：
```
总目标          总达成进度        同比
¥230,910,000    12.1%           -30%
(18px)          (36px)          (36px)
```
**问题**：中间列 36px 的大号百分比跟左右列字号落差过大，视觉重心偏移影响阅读流。

### 问题 2：信息层级混乱

当前卡片有清晰的信息层级：
- 第一级：GMV 金额（最大字）
- 第二级：目标值 + 达成率（辅助指标）
- 第三级：进度条（视觉占比）

直接塞入第三个指标会让第二行从左到右出现"绝对值 / 百分比 / 百分比"的混合密度，破坏卡片呼吸感。

### 问题 3：同比是"负向指标"但无色彩语义

同比 `-30%` 表示下降，如果只是一串文字，用户无法快速判断好坏。银行/金融类卡片通常用红绿色彩区分正负。

## 推荐方案：2 行布局

```
┌─ total-card-shell ─────────────────────────────────────┐
│ 北京时间：18:15:21                                      │
│ ┌─ total-card ─────────────────────────────────────────┐│
│ │ 全渠道实时GMV                          · 实时模式    ││
│ │ ¥280,530,340                                         ││
│ │                                                      ││
│ │  总目标          总达成进度         同比  vs 同期     ││
│ │  ¥230,910,000    12.1%     │    -30.0%               ││
│ │                                                      ││
│ │ ██████░░░░░░░░░░░░░░░░░░                             ││
│ └──────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────┘
```

### 设计要点

#### 布局
- 使用 `grid-template-columns: 1fr auto 1fr` 三列
- 中列放一个竖线分隔符 `│`（视觉分割达成进度和同比）
- 三列标签：`总目标` / `总达成进度` / `同比 vs 同期`

#### 字号
- 达成进度 36px 900 → **改为 28px 900**（给同比留空间）
- 同比 28px 900，与达成进度同号等重
- 分隔符 24px，color: var(--line)

#### 色彩语义
- 同比正值（增长）→ `var(--accent)` 青色 `#18d9d2`
- 同比负值（下降）→ `var(--bad)` 红色 `#ff4d68`  
- 同比无数据 → `var(--muted)` 灰色

#### 进度条
- 不变，保持全宽在底部

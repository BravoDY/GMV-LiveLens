# Tasks
- [ ] Task 1: 固化逐店迁移分组
  - [ ] 根据最新实库审计结果，逐店整理当前目录、期望目录、模式和风险标签
  - [ ] 将店铺分为“保留旧目录”“人工确认后再决定”“确认后优先迁移”三组
  - [ ] 标明每组的判定依据，避免只凭目录名下结论

- [ ] Task 2: 设计人工确认流程
  - [ ] 为需要人工确认的店铺整理统一检查项：商家后台店铺名、主体身份、业务看板归属
  - [ ] 明确哪些确认结果代表“继续保留旧目录”
  - [ ] 明确哪些确认结果代表“进入迁移流程”

- [ ] Task 3: 输出单店迁移执行方案
  - [ ] 为确认后优先迁移的店铺定义迁移前备份与停用条件
  - [ ] 为单店迁移整理执行顺序、风险提示和回滚关注点
  - [ ] 为迁移后整理登录、页面绑定、预览、采集的验收标准

- [ ] Task 4: 输出最终交付清单
  - [ ] 汇总逐店结论、人工确认步骤和迁移执行方案
  - [ ] 明确哪些店当前不建议迁移，哪些店需要优先检查
  - [ ] 明确本次方案只提供执行依据，不直接改库或切换目录

# Task Dependencies
- Task 2 depends on Task 1
- Task 3 depends on Task 1 and Task 2
- Task 4 depends on Task 1, Task 2 and Task 3
